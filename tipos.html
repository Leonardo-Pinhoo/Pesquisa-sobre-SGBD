Tipos de SGBD

SGBD Relacional

Um SGBD relacional é um sistema de gerenciamento de banco de dados que armazena dados em tabelas (ou relações) compostas por linhas e colunas. Cada linha representa um registro, e cada coluna representa um atributo do registro. O modelo relacional é baseado na teoria dos conjuntos e na álgebra relacional, permitindo operações como seleção, projeção, junção e união para manipular os dados.

Exemplos populares de SGBDs relacionais incluem MySQL, PostgreSQL, Oracle Database e Microsoft SQL Server.

SGBD Não Relacional (NoSQL)

Um SGBD não relacional, também conhecido como NoSQL (Not Only SQL), é um sistema de gerenciamento de banco de dados que não utiliza o modelo relacional tradicional. Em vez disso, ele pode armazenar dados em formatos variados, como documentos, grafos, colunas ou pares chave-valor. Os SGBDs NoSQL são projetados para lidar com grandes volumes de dados, alta escalabilidade e flexibilidade na estrutura dos dados.

Exemplos populares de SGBDs NoSQL incluem MongoDB (documento), Cassandra (coluna), Redis (chave-valor) e Neo4j (grafo).
